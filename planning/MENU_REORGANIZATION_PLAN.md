# 메뉴 재정리 계획

## 현재 메뉴 구조 분석

### 1. 설정 메뉴
- 환경설정
- MCP 확장 임포트
- MCP 서버 관리
- 설정 파일 경로 설정
- 대화 기록 초기화
- 유저 프롬프트 설정
- 세션 패널 표시
- 토큰 사용량 표시
- 글래스모피즘 효과
- 레이아웃 초기화

### 2. RAG 메뉴
- RAG 관리
- RAG 설정
- RAG 테스트

### 3. 보안 메뉴
- 로그아웃
- 보안 상태

## 문제점
1. **설정 메뉴 과부하**: 너무 많은 항목이 한 메뉴에 집중
2. **논리적 그룹화 부족**: 관련 기능들이 분산
3. **계층 구조 부재**: 모든 항목이 동일 레벨
4. **일관성 없는 아이콘**: 일부만 이모지 사용

## 재정리 방안

### 새로운 메뉴 구조

```
📁 파일
  ├─ 새 세션
  ├─ 세션 열기
  ├─ 세션 저장
  ├─ 세션 내보내기
  ├─ ───────────
  └─ 종료

⚙️ 설정
  ├─ 환경설정
  ├─ 설정 파일 경로
  ├─ ───────────
  └─ 유저 프롬프트

🔌 확장 기능
  ├─ MCP 서버 관리
  ├─ MCP 확장 임포트
  ├─ ───────────
  ├─ RAG 관리
  └─ RAG 설정

🎨 보기
  ├─ 세션 패널 표시
  ├─ 토큰 사용량 표시
  ├─ ───────────
  ├─ 글래스모피즘 효과
  └─ 레이아웃 초기화

🔒 보안
  ├─ 보안 상태
  ├─ ───────────
  └─ 로그아웃

🛠️ 도구
  ├─ 대화 기록 초기화
  ├─ ───────────
  └─ RAG 테스트

❓ 도움말
  ├─ 사용 가이드
  ├─ 단축키 목록
  ├─ ───────────
  ├─ 버전 정보
  └─ 라이선스
```

## 변경 사항 요약

### 신규 메뉴
- **파일**: 세션 관리 기능 통합
- **확장 기능**: MCP + RAG 통합
- **보기**: UI 표시 옵션 통합
- **도구**: 유틸리티 기능 통합
- **도움말**: 사용자 지원 정보

### 이동된 항목
- MCP 관련 → 확장 기능
- RAG 관련 → 확장 기능
- UI 토글 → 보기
- 대화 기록 초기화 → 도구
- RAG 테스트 → 도구

### 삭제 예정
- 기존 "설정" 메뉴 (항목들을 적절히 분산)
- 기존 "RAG" 메뉴 (확장 기능으로 통합)

## 구현 우선순위

### Phase 1: 핵심 재구성
1. 파일 메뉴 생성
2. 설정 메뉴 간소화
3. 확장 기능 메뉴 생성 (MCP + RAG)

### Phase 2: UI 개선
4. 보기 메뉴 생성
5. 도구 메뉴 생성
6. 아이콘 일관성 확보

### Phase 3: 사용자 지원
7. 도움말 메뉴 생성
8. 단축키 통합
